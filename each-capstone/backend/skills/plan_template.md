# Weekly Plan — Output Template

The Planning Agent MUST follow this exact structure.
Do not invent a different format. Do not omit sections.
Both Arabic (AR) and English (EN) versions are required in every plan.

## Rules for the Planning Agent

1. **Never skip a day.** All 5 days must be present.
2. **Never skip the AR section.** Both AR and EN are required.
3. **Goals must be measurable.** Bad: "improve focus". Good: "complete 2 math tasks with one break".
4. **Activities must have durations.** Bad: "reading activity". Good: "paired reading — 15 min".
5. **Success indicator must be observable.** Bad: "student does better". Good: "student completes task with ≤2 redirections".
6. **Strategies must match the diagnosis skill file.** Do not invent strategies not grounded in the loaded skill.
7. **Teacher notes must be practical.** Write as if handing this to the teacher at 7am before school starts.
8. **Version number** must increment from the previous plan's version for the same student + week.

---

## Filling in the Header Fields

| Field | Source |
|---|---|
| `name_ar` / `name_en` | `students` table |
| `diagnosis` | `students.diagnosis` |
| `week_start_date` | Next Monday from today |
| `week_end_date` | Friday of same week |
| `teacher_name` | `users` table via `sessions.teacher_id` |
| `version` | `max(weekly_plans.version) + 1` for this student + week |
